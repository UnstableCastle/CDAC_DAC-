C PROGRAMMING LANGUAGE

High-Level Language Features:

Procedural Oriented Programming: Uses modular sub-programs, functions, and procedures

Syntax close to English

Case-sensitive

32 keywords

Compiler-based

General-Purpose Language

Used to develop: web apps, software, operating systems, antivirus, games, etc.

Low-Level Language

Machine-understandable code (binary: combinations of 0s & 1s)

Compiler Role

Translates high-level language (C) to low-level machine code

Acts as a translator between programmer and machine

Key Concepts:

Aspect	Details

OS Limitation : 	Traditional OS doesn't natively support OOP (Object-Oriented Programming)
Compiler Dependency :	Machine-dependent (tied to specific CPU + OS)



C File Extension	.c files

Execution Flow	.c file → Compiler → Linker → OS Loader → RAM → CPU


Memory Addressing:

Logical Addresses: Generated by Compiler + Linker

Physical Addresses: Managed by Operating System

Direct Memory Access (DMA)

Hardware feature that handles memory allocation and management directly (bypasses CPU)

Program Execution Process:


.c source file 
 
   ↓ (Compiler)

    ↓ (Linker)

Executable file

    ↓ (OS Loader)

RAM 

    ↓ (CPU)

Machine execution
