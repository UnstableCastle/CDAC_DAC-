WHAT IS REACT?
--------------------------------------------------------------------
- React is a JavaScript library
- Used to create scalable, attractive UI (User Interface)
- Used for building Single Page Applications (SPA)
- Developed & maintained by META / Facebook
- Based on Component Architecture
- Latest Version: React 19
- flow of data is UNIDIRECTIONAL 

--------------------------------------------------------------------
WHAT IS COMPONENT?
--------------------------------------------------------------------
- A component is a reusable part of a React application
- Create once, use multiple times
- Helps in modular and maintainable code

--------------------------------------------------------------------
TECHNOLOGIES REQUIRED
--------------------------------------------------------------------
- HTML
- CSS
- JavaScript (ES5, ES6 - ECMAScript)
- OOP Concepts

--------------------------------------------------------------------
SOFTWARE REQUIRED
--------------------------------------------------------------------
- Node.js
- npm (Node Package Manager)
- CRA (Create React App)
- IDE (VS Code, etc.)

--------------------------------------------------------------------
HOW TO CREATE REACT PROJECT
--------------------------------------------------------------------

1. USING NPX (Recommended)
--------------------------------
S1 -> npx create-react-app <appname>
S2 -> cd <appname>
S3 -> npm start

✔ No need to install globally

--------------------------------
2. USING NPM
--------------------------------
S1 -> npm install -g create-react-app
S2 -> create-react-app <appname>
S3 -> cd <appname>
S4 -> npm start

✖ Requires global installation

--------------------------------
3. USING VITE
--------------------------------
- Faster & lightweight setup
- Minimal project size
- Uses .jsx files

--------------------------------------------------------------------
TYPES OF COMPONENTS
--------------------------------------------------------------------

1. FUNCTION COMPONENT
----------------------
- Used mainly for UI design
- Simple and modern approach

2. CLASS COMPONENT
----------------------
- Used for complex logic
- Uses OOP concepts

--------------------------------------------------------------------
JSX (JavaScript XML)
--------------------------------------------------------------------
- Allows writing HTML inside JavaScript
- Makes code readable & easy

Rules:
- Must return a single parent element
- Parent should be block-level (div, p, etc.)

--------------------------------------------------------------------
PROPS
--------------------------------------------------------------------
- Props = Properties (like function parameters)
- Used to pass data between components
- Returns object type

Access:
- Function Component → props.key
- Class Component → this.props.key

--------------------------------------------------------------------
STATE
--------------------------------------------------------------------
- Used to manage dynamic data
- Returns object type

Class Component:
- Defined inside constructor
- Access → this.state.key

Function Component:
- Using useState Hook

--------------------------------------------------------------------
setState()
--------------------------------------------------------------------
- Used to update state in class components
- Built-in React method

--------------------------------------------------------------------
RENDERING
--------------------------------------------------------------------
RENDER:
- Display component output on browser

CONDITIONAL RENDERING:
----------------------
1. if-else
2. Element as variable
3. Ternary Operator → (condition) ? true : false
4. Short Circuit (&&) → for null checking

--------------------------------------------------------------------
COMPONENT INTERACTION
--------------------------------------------------------------------
- Parent → Child data flow using props
- Passing state data to child component

--------------------------------------------------------------------
HOOKS (React 16+)
--------------------------------------------------------------------
- Allow using class features in function components

Rules:
- Only used in function components
- Cannot be used inside conditions
- Must be used at top level

--------------------------------------------------------------------
useState() HOOK
--------------------------------------------------------------------
- Used to manage state in function components

Syntax:
const [state, setState] = useState(initialValue)

Returns:
1. Current State
2. Function to update state

--------------------------------------------------------------------
VIRTUAL DOM
--------------------------------------------------------------------
- React creates a virtual copy of DOM in memory
- Updates only changed parts
- Makes React faster than Angular

        useEffect CASES (IMPORTANT)
==================================================

CASE 1: NO DEPENDENCY ARRAY
--------------------------------------------------
useEffect(() => {
    setCount(count + 1);
});

- Runs on EVERY render
- Can cause INFINITE LOOP if state is updated inside
-
-
-

WHY?
- setCount triggers re-render
- useEffect runs again → loop continues

⚠️ Use carefully


CASE 2: EMPTY DEPENDENCY ARRAY []
--------------------------------------------------
useEffect(() => {
    // code
}, []);

- Runs ONLY ONCE after first render
-
-
-
- Used to achieve:
    COMPONENT DID MOUNT


SYNTAX:
--------------------------------------------------
useEffect(() => {
    // logic
}, []);


EXAMPLE:
--------------------------------------------------
useEffect(() => {
    console.log("Component Mounted");
}, []);



CASE 3:  DEPENDENCY ARRAY VALUES PASSES[]
--------------------------------------------------
useEffect(() => {
   setCount(count+1);
},[age]);

Used to achieve:
    COMPONENT DID MOUNT



CASE 4 : When dependency valu pass bs blank array and in insied callback function we use cleartime() method

when dependency 





KEY DIFFERENCE
--------------------------------------------------
NO ARRAY        → runs every render ❌
EMPTY ARRAY []  → runs once (mount) ✅


EXTRA POINT (VIMP)
--------------------------------------------------
- Empty dependency array = safest for API calls
- Prevents unnecessary re-renders
- Avoid infinite loops

==================== REACT FORM + CRUD + ROUTING ====================

1) EVENT HANDLING (onChange)
---------------------------------------------------------------------
const handleChange = (event) => {
  console.log(event.target.name);
  console.log(event.target.value);
};

event.target.name   -> input field name
event.target.value  -> input field value


2) USESTATE WITH OBJECT
---------------------------------------------------------------------
const [data, setData] = useState({
  name: "",
  price: ""
});


3) UPDATE STATE (SPREAD OPERATOR)
---------------------------------------------------------------------
const handleChange = (event) => {
  setData({
    ...data,
    [event.target.name]: event.target.value
  });
};

...data  -> keeps old values
[name]   -> updates only changed field


4) DESTRUCTURING
---------------------------------------------------------------------
const { name, value } = event.target;

setData({
  ...data,
  [name]: value
});


5) FORM SUBMIT
---------------------------------------------------------------------
const handleSubmit = (event) => {
  event.preventDefault();   // stop refresh
  console.log(data);
};


6) POST DATA (ADD)
---------------------------------------------------------------------
axios.post("URL", data)
  .then(res => {
    alert("Data Added Successfully");
  })
  .catch(err => console.log(err));


7) NAVIGATION (useNavigate)
---------------------------------------------------------------------
import { useNavigate } from "react-router-dom";

const navigate = useNavigate();

navigate("/dashboard");


8) PASS ID IN URL
---------------------------------------------------------------------
<Link to={`/update/${id}`}>Edit</Link>


9) GET ID FROM URL
---------------------------------------------------------------------
import { useParams } from "react-router-dom";

const { id } = useParams();


10) FETCH SINGLE DATA (GET)
---------------------------------------------------------------------
useEffect(() => {
  axios.get(`URL/${id}`)
    .then(res => {
      setData(res.data);
    });
}, []);


11) UPDATE DATA (PUT)
---------------------------------------------------------------------
axios.put(`URL/${id}`, data)
  .then(res => {
    alert("Data Updated Successfully");
    navigate("/dashboard");
  });


12) COMPLETE FLOW
---------------------------------------------------------------------
INPUT → onChange → state update
       ↓
FORM SUBMIT → preventDefault()
       ↓
API CALL (POST / PUT)
       ↓
SUCCESS → navigate("/dashboard")


==================== KEY POINTS ====================
- Controlled components (state controls input)
- event.target → name & value
- spread operator (...) for safe updates
- useParams → get id
- useNavigate → redirect
- axios → API calls (GET, POST, PUT)
